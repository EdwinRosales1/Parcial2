TECNOLOGIAS WEB APIS

¿QUÉ ES UNA API?

Una API (interfaz de programación de aplicaciones), es un conjunto de reglas definidas que permiten que diferentes aplicaciones se comuniquen entre sí. Actúa como una capa intermedia que procesa las transferencias de datos entre sistemas, permitiendo a las empresas abrir sus datos y funcionalidades de aplicaciones a desarrolladores externos, socios comerciales y departamentos internos dentro de sus empresas.

URI

Identificador de recursos uniforme, es una cadena de caracteres que se utiliza para identificar recurso en la web de manera única.

Existen dos tipos de URI

URL:identifica un recurso indicando donde esta y como alcanzarlo.
URN:identifica un recurso mediante su nombre.

Clasificacion de los codigos de estado:
  1. xx Respuesta infomativa
  2. xx Respuesta de Exito
  3. xx Redirecciones
  4. xx Errores de cliente
  5. xx Errores de servidor

¿QUÉ ES EL HTTP?

Significa Protocolo de Transferencia de Hipertexto (Hypertext Transfer Protocol) es el lenguaje estándar utilizado para crear y diseñar páginas web. Es un lenguaje de marcado que define la estructura y el contenido de una página web mediante el uso de elementos y etiquetas. HTML permite a los desarrolladores web organizar y presentar información en un formato que puede ser interpretado por los navegadores web..

METODOS DE HTTP
  • GET:Solicita una representación del recurso especificado. Solo debe recuperar datos y no debe tener ningún otro efecto.
  • POST:Envía datos al servidor para ser procesados. Los datos se incluyen en el cuerpo del mensaje. Se utiliza para enviar información para ser procesada por un recurso identificado por la URL.
  • PUT:Reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la solicitud.
  • DELETE:Elimina el recurso especificado.
  • PATCH:Aplica modificaciones parciales a un recurso.
  • HEAD:Solicita una respuesta similar a GET, pero sin el cuerpo de la respuesta. Se utiliza para obtener metadatos sobre el recurso sin solicitar el cuerpo de la respuesta.
  • OPTIONS:Utilizado para describir las opciones de comunicación para el recurso de destino.
  • TRACE:Realiza una prueba de bucle de retorno de mensaje a lo largo de la ruta al recurso de destino.
SOAP

Protocolo que Permite que aplicaciones en diferentes plataformas se comuniquen entre sí de manera independiente del lenguaje de programación y el sistema operativo utilizados.

Ventajas:
  1. Legibilidad: Debido a su formato XML, facilita la depuración y el monitoreo.
  2. Amplia Adopcion:es ampliamente adoptado en la industria y es compatible con numerosas tecnologías y plataformas.
Desventajas:
  1. Complejidad:Su flexibilidad puede llevar a una mayor complejidad en la implementación y la comprensión.
  2. Overhead:El formato XML agregar sobrecarga en términos de tamaño de mensaje y procesamiento, lo que puede hacer que SOAP sea menos eficiente.
Beneficios:
  1. Seguridad: Implementa las medidas de seguridad sólidas, como SSL/TLS, para garantizar la confidencialidad y la autenticación de los datos.
  2. Robustez: estructura basada en XML lo hace resistente a cambios en la plataforma y el lenguaje de programación.
¿QUÉ ES UN API REST?

Una API REST es una API que se ajusta a los principios de diseño de REST, o el estilo de arquitectura de transferencia de estado representacional. Por esta razón, las API REST a veces se denominan API RESTful. REST proporciona un nivel relativamente alto de flexibilidad y libertad para los desarrolladores. Esta flexibilidad es solo una de las razones por las que las API REST han surgido como un método común para conectar componentes y aplicaciones en una arquitectura de microservicios.

Principios de diseño de REST
  1. Interfaz uniforme. Todas las solicitudes de API para el mismo recurso deben tener el mismo aspecto, sin importar de dónde provenga la solicitud. La API REST debe garantizar que el mismo dato, como el nombre o la dirección de correo electrónico de un usuario, pertenezca a un solo identificador uniforme de recursos (URI).
  2. Desacoplamiento cliente-servidor. En el diseño de API REST, las aplicaciones de cliente y servidor deben ser completamente independientes entre sí. La única información que debe conocer la aplicación del cliente es el URI del recurso solicitado; no puede interactuar con la aplicación del servidor de ninguna otra manera.
  3. Sin estado. Las API REST no tienen estado, lo que significa que cada solicitud debe incluir toda la información necesaria para procesarla.
  4. Capacidad de caché. Cuando sea posible, los recursos deben almacenarse en la memoria caché en el lado del cliente o del servidor. Las respuestas del servidor también deben contener información sobre si se permite el almacenamiento en caché para el recurso entregado
  5. Arquitectura del sistema en capas. En las API REST, las llamadas y respuestas pasan por diferentes capas. Como regla general, no asuma que las aplicaciones cliente y servidor se conectan directamente entre sí.
  6. Código bajo demanda. Las API REST generalmente envían recursos estáticos, pero en ciertos casos, las respuestas también pueden contener código ejecutable (como los subprogramas de Java).
JWT

Significa "JSON Web Token". Es un estándar abierto (RFC 7519) que define un formato compacto y autónomo para la transmisión segura de información entre dos partes como un objeto JSON. Este token se utiliza comúnmente para autenticar usuarios y compartir información entre un cliente y un servidor de forma segura. Un JWT consta de tres partes separadas por puntos: el encabezado, la carga útil y la firma. Un JWT consta de tres partes separadas por puntos: el encabezado, la carga útil y la firma. El encabezado contiene el tipo de token y el algoritmo de firma utilizado, la carga útil contiene la información que se desea transmitir y la firma se utiliza para verificar que el mensaje no ha sido modificado en el camino. Los JWT son ampliamente utilizados en aplicaciones web y servicios API como una forma de autenticación y autorización.
